<Note>
All Lambda environment variables are read from SSM at cold start. Never set these manually — the wizard and SAM template handle them.
</Note>

## Variable reference

| Variable                       | Source           | Description                                  |
| ------------------------------ | ---------------- | -------------------------------------------- |
| `GITHUB_APP_ID`                | SSM SecureString | Numeric GitHub App ID                        |
| `GITHUB_PRIVATE_KEY`           | SSM SecureString | PEM-encoded RSA private key                  |
| `GITHUB_WEBHOOK_SECRET`        | SSM SecureString | HMAC secret for payload validation           |
| `GITHUB_APP_SLUG`              | SSM String       | GitHub App slug (used in installation URLs)  |
| `BEDROCK_MODEL_ID`             | SAM config       | Default model for all agents                 |
| `BEDROCK_LIGHT_MODEL_ID`       | SAM config       | Light model for summary/low-complexity tasks |
| `BEDROCK_REGION`               | SAM config       | AWS region for Bedrock API calls             |
| `DYNAMODB_INSTALLATIONS_TABLE` | SAM output       | Installations DynamoDB table name            |
| `DYNAMODB_REVIEWS_TABLE`       | SAM output       | Reviews DynamoDB table name                  |
| `SQS_QUEUE_URL`                | SAM output       | SQS FIFO queue URL                           |
| `STAGE`                        | SAM config       | Deployment stage (`prod`, `staging`, etc.)   |
| `VERSION`                      | Build            | Deployed version (injected at build time)    |

<Warning>
SSM SecureString parameters are encrypted at rest and in transit. The Lambda function decrypts them at cold start and caches the values for the lifetime of the execution environment.
</Warning>

## Source categories

<AccordionGroup>
  <Accordion title="SSM SecureString">
    These values are stored as encrypted SSM parameters under the path `/mergewatch/<stage>/`. They contain sensitive credentials and are never logged or exposed in CloudFormation outputs.
  </Accordion>
  <Accordion title="SAM config">
    These values are set in `samconfig.toml` and passed as CloudFormation parameters during deployment. They are visible in the CloudFormation console.
  </Accordion>
  <Accordion title="SAM output">
    These values are generated by CloudFormation during stack creation and automatically injected into the Lambda environment. They reference the physical resource names of deployed infrastructure.
  </Accordion>
  <Accordion title="Build">
    These values are injected at build time by the CLI or CI/CD pipeline. They are baked into the deployment artifact.
  </Accordion>
</AccordionGroup>
